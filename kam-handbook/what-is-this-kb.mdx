This is the internal knowledge base for eGora **Key Account Managers (KAMs)**. A KAM owns a catering
request end-to-end: from an inbound request, through matching caterers and sending an offer, to a booked
order delivered and invoiced.

<Note>
  **Real-testing phase.** Pages are grounded in eGora's actual system (real statuses, the real
  post-booking checklist). Where something is a **working convention** rather than enforced by the
  system, the page says so. If a process isn't formalized, this KB says that too — it does not invent
  one.
</Note>

## Two surfaces, one source

This KB is read both by **you on the web** and by the **eGora KAM Assistant** (Claude) via MCP — so you
can either read it here or just ask the assistant. The assistant is told to treat this KB as the
**authoritative** source for "how does X work" questions.

<Warning>
  The assistant relays what's in this KB. If a page is wrong, the assistant will confidently repeat it —
  so flag inaccuracies (a Slack `@mention` or a PR) rather than letting them stand.
</Warning>
