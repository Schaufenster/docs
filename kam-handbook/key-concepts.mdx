Core vocabulary, grounded in eGora's codebase. Where the business meaning beyond the code isn't defined,
it's marked.

## Objects

<CardGroup cols={2}>
  <Card title="CateringRequest" icon="inbox">
    The inbound request/lead: headcount, date/time, budget, B2B flag, lead source, assigned KAM. One
    request can lead to one or more offers.
  </Card>
  <Card title="CateringOffer" icon="file-invoice">
    An offer/quote on a request. Marked *sent* via the `sent_to_customer_at` timestamp (not a status).
    An accepted offer becomes an Order.
  </Card>
  <Card title="Order" icon="receipt">
    The booked/paid transaction from an accepted offer. Carries the order status, the Stornogarantie line
    item, and the [post-booking checklist](/kam-handbook/post-booking-checklist).
  </Card>
  <Card title="Customer vs Company (B2B)" icon="building">
    A request has a customer and an `is_b2b` flag; companies are a separate B2B entity (billing profiles,
    company users, event formats). *The precise distinction beyond the B2B flag is not spelled out in
    code — confirm with the team if it matters.*
  </Card>
</CardGroup>

## Caterer side

- **Retailer = caterer** — a catering business on the marketplace.
- **Branch = a caterer location** — a retailer can have several branches, each with its own opening
  hours, minimum order value and delivery support.

## Key terms

- **Stornogarantie (cancellation guarantee)** — a tiered line item auto-added to offers/orders (SKU
  prefix `EGORA-STORNOGARANTIE`), in four tiers (T1–T4) that scale with order value. On an **offer** it's
  added/updated automatically unless the KAM opts out (`exclude_stornogarantie`); on an **order** it only
  **resizes**, never auto-adds. The opt-out carries from offer to order.
- **KAM** — Key Account Manager: owns a request end-to-end.
- **GMV** — gross merchandise value (booked order value). The internal KPI dashboards exclude Checkout,
  Storno/Cancelled and Payment-failed orders from booked-GMV totals.

## Where eGora operates

**Berlin and München only** — the only cities seeded in the system. Timezone: Europe/Berlin.

<Warning>
  The assistant cannot see customer **personal data** (names, emails, phones, addresses, free-text
  notes) — those are redacted in the data connector. It won't invent them.
</Warning>
