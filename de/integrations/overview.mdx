Die KB ist **eine git-basierte Quelle** (`Schaufenster/docs` → Mintlify). Sie wird von zwei Arten von Consumern gelesen
und durch einige reviewte Inflows gespeist.

## Wer sie liest

<CardGroup cols={2}>

  </Card>
  <Card title="Claude (via MCP)" icon="robot">
    Der eGora KAM Assistant liest die KB über Mintlifys MCP-Server — siehe
    [Claude anbinden](/de/integrations/claude-setup).
  </Card>
</CardGroup>

## Was Claude füttert

Claude wird durch **vier** Quellen geerdet, jede für einen anderen Zweck:

| Quelle | Wofür sie da ist |
| --- | --- |
| **Mintlify KB MCP** | Stabiles, offizielles Wissen (diese KB) |
| **egora-data MCP** | Live-Geschäftsdaten (Bestellungen, Anfragen, KPIs) — siehe [egora data](/de/integrations/egora-mcp) |
| **Notion** | Arbeitsdokumente, live gelesen — siehe [Notion](/de/integrations/notion) |
| **Skills** | Power-User-Verhalten in Claude Code / Cowork |

Faustregel: **KB MCP** für "Wie funktioniert X", **egora-data** für "Wie sind die aktuellen Zahlen",
**Notion** für "Was ist der neueste Entwurf".

## Was sie aktuell hält (Authoring-Inflows)

Jede Änderung landet als **reviewbarer Pull Request** — nichts wird stillschweigend committet.
