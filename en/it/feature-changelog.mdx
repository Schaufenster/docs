The core problem this knowledge base addresses: **features go live without being communicated** — to
the people they affect. This page shows a repeatable pattern, so a feature isn't considered "done"
until its user-facing change is written down for every audience.

<Note>
  The entries below are **illustrative examples** to show the format — not real release notes.
</Note>

## The pattern

Every shipped feature gets a changelog entry that answers three questions:

<Steps>
  <Step title="What changed (one sentence)">
    Plain language, no jargon — the behavior a user would notice.
  </Step>
  <Step title="Who's affected">
    Tag the audiences: **KAM**, **customers**, **caterers**. If an audience is affected, the entry
    has to say what *they* should do differently.
  </Step>
  <Step title="Where to find more">
    Link to the relevant page in the customer or caterer tab so the announcement and the reference
    docs stay in sync.
  </Step>
</Steps>

## Changelog

<Update label="2026-06-25" description="OrderHub: invoice action verifies tax rates">
  The four KAM checklist actions in OrderHub (**Order Confirmation → Customer**, **Order Confirmation
  → Caterer**, **Delivery Sheet → Caterer**, **Invoice → Customer**) now behave identically to the
  Actions dropdown. In particular, **Invoice → Customer** no longer shows a green success toast when
  the cart has unconfirmed tax-rate issues without the Stripe invoice actually being created —
  instead, a confirmation lists the affected line items, and the invoice is only created after you
  confirm.

  <CardGroup cols={3}>
    <Card title="KAM" icon="headset">
      When you click **Invoice → Customer** in the checklist and a tax notice appears, review the
      listed line items. Cancel sends nothing; OK creates the invoice anyway. The previously
      misleading green success toasts are gone.
    </Card>
    <Card title="Customers" icon="user">
      Not affected.
    </Card>
    <Card title="Caterers" icon="utensils">
      Not affected.
    </Card>
  </CardGroup>
</Update>
