The KAM backend is the **Laravel Nova administration** (at `/nova`). Caterers and customers use a
separate Filament backend – a KAM works **exclusively in Nova**.

What you see depends on your role. With the **`kam`** role, **eight menu sections** are visible (the
admin/configuration areas stay hidden, see below). This reference covers exactly those eight sections –
anchored to the actual menu definition in code.

## The eight KAM sections

## Menu map

| Section | Contains (resources & tools) |
| --- | --- |
| **Dashboard** | Main dashboard (order KPIs, cart, delivery, voucher) |
| **KAM Inbox** | Per-KAM activity feed *(feature flag)* |
| **Catering Operations** | CateringRequest · Request Email Parser · Caterer Matching Tool · Catering offer tool (manual) · CateringOffer · Order · InvoiceDunningReminder · PrecomputedMenuProposal · TextSnippet |
| **Feedback & Surveys** | SurveyInstance · SurveyAnswer · Survey insights dashboard |
| **Catalog & Suppliers** | Retailer · Branch · MonthlyRetailerBill · PayoutAdjustment · Product · Package · Productvariant · Voucher |
| **Users & Customers** | User · Company |
| **Smart Profiles** | BillingProfile · EventFormat |
| **Performance & Insights** | Request / Marketing / Finance / Misc KPIs · LeadFormProgress · GoogleAd · LeadSource |

## What a KAM deliberately does **not** see

These areas are only visible to admins – reach out to the team if you need something here:

<Note>
  **Catering Config**, **Delivery Config**, **Content Management**, **Reference Data** (master data / taxonomies),
  **System Admin** (roles, permissions, imports), **Email & Communication** (email templates / A-B tests) and
  **Accounting & Finance** (dunning exceptions).
</Note>

<Tip>
  The end-to-end flow of a request lives in [Request lifecycle](/en/kam-handbook/request-lifecycle); the
  terms (Retailer, Branch, Stornogarantie, GMV) are in the [glossary](/en/kam-handbook/key-concepts).
</Tip>
