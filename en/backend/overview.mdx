The KAM backend is the **Laravel Nova administration** (at `/nova`). Caterers and customers use a
separate Filament backend – a KAM works **exclusively in Nova**.

What you see depends on your role. With the **`kam`** role, **eight menu areas** are visible (the
admin/configuration areas stay hidden, see below). This reference describes exactly those eight areas –
anchored in the actual menu definition in code.

## The eight KAM areas

<CardGroup cols={2}>
  <Card title="Dashboard" icon="chart-bar">
    The home page with aggregated order/cart/delivery metrics.
  </Card>
  <Card title="KAM Inbox" icon="inbox-full" href="/backend/tools">
    Your unified activity feed (email, chat, system events). Enabled via a feature flag.
  </Card>
  <Card title="Catering Operations" icon="utensils" href="/backend/catering-operations">
    Your day-to-day: requests, offers, orders, matching and menu tools.
  </Card>
  <Card title="Feedback & Surveys" icon="star" href="/backend/customers-profiles">
    Customer/caterer surveys and the survey insights.
  </Card>
  <Card title="Catalog & Suppliers" icon="store" href="/backend/catalog-suppliers">
    Caterers (Retailers), locations (Branches), billing, products, packages, vouchers.
  </Card>
  <Card title="Users & Customers" icon="users" href="/backend/customers-profiles">
    Platform users and B2B companies.
  </Card>

  </Card>
  <Card title="Performance & Insights" icon="gauge" href="/backend/performance-insights">

  </Card>
</CardGroup>

## Menu map

| Area | Contains (resources & tools) |
| --- | --- |
| **Dashboard** | Main dashboard (order KPIs, cart, delivery, voucher) |
| **KAM Inbox** | Per-KAM activity feed *(feature flag)* |
| **Catering Operations** | CateringRequest · Request Email Parser · Caterer Matching Tool · Catering offer tool (manual) · CateringOffer · Order · InvoiceDunningReminder · PrecomputedMenuProposal · TextSnippet |
| **Feedback & Surveys** | SurveyInstance · SurveyAnswer · Survey insights dashboard |
| **Catalog & Suppliers** | Retailer · Branch · MonthlyRetailerBill · PayoutAdjustment · Product · Package · Productvariant · Voucher |
| **Users & Customers** | User · Company |
| **Smart Profiles** | BillingProfile · EventFormat |
| **Performance & Insights** | Request / Marketing / Finance / Diverse KPIs · LeadFormProgress · GoogleAd · LeadSource |

## What a KAM intentionally **doesn't** see

These areas are only visible to admins – reach out to the team if you need something here:

<Note>
  **Catering Config**, **Delivery Config**, **Content Management**, **Reference Data** (master data /
  taxonomies), **System Admin** (roles, permissions, imports), **Email & Communication** (email templates
  / A-B tests) and **Accounting & Finance** (dunning exceptions).
</Note>

<Tip>
  The end-to-end flow of a request is in the [request lifecycle](/kam-handbook/request-lifecycle); the
  terminology (Retailer, Branch, Stornogarantie, GMV) is in the [glossary](/kam-handbook/key-concepts).
</Tip>
