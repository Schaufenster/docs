The analytics area. The four KPI menu items all link to the **same** catering KPI page
(`/catering-kpi-page`), just anchored to different sections.

## KPI page

| Menu item | Jump anchor | Content |
| --- | --- | --- |
| **Request KPIs** | `#conversion-rate-kpis` | Conversion funnel (converted, completed, lost …) |
| **Marketing KPIs** | `#google-ads-kpis` | Google Ads funnel & cohorts |
| **Finance KPIs** | `#finance-kpis` | Revenue, commission, payouts, taxes |
| **Diverse KPIs** | `#backend-visit-rate-kpis` | Backend usage, caterer/customer interaction |

<Note>
  The KPI page defines **~109 canonical metric tiles** in 12 sections. It is the source of truth for
  "what is our X?". The full list is in the [KPI catalog](/it/kpi-catalog); the Claude assistant reads
  the same values server-side via the `get_catering_kpi` MCP function.
</Note>

## Funnel & attribution resources

<CardGroup cols={2}>
  <Card title="LeadFormProgress" icon="filter">

  </Card>

<Warning>
  There are two GMV perspectives (by **request date** vs. by **booking/item date**), plus a
  transaction-based variant. Always compare the same definition – the KPI page labels it per tile.
</Warning>
