This is the internal knowledge base for eGora **Key Account Managers (KAMs)**. A KAM owns a
catering request end-to-end: from the incoming request, through matching with caterers and sending an
offer, all the way to a booked Order that has been delivered and invoiced.

<Note>
  **Real-testing phase.** The pages are anchored in eGora's real system (actual statuses, the actual
  post-booking checklist). Where something is a **working convention** and not system-enforced, the page
  says so. If a process isn't formalized, this KB says so too — it doesn't invent one.
</Note>

## Two surfaces, one source

This KB is read both by **you on the web** and by the **eGora KAM Assistant** (Claude) via MCP — so you
can either read it here or just ask the assistant. The assistant is instructed to treat this KB as the
**authoritative** source for "how does X work" questions.

<Warning>
  The assistant repeats what this KB says. If a page is wrong, the assistant will confidently repeat
  that — so report inaccuracies (via Slack `@mention` or PR) instead of leaving them in place.
</Warning>
