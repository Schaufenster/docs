Core vocabulary, anchored in the eGora codebase. Where the business meaning beyond the code is not
defined, that's flagged.

## Objects

<CardGroup cols={2}>
  <Card title="CateringRequest" icon="inbox">
    The incoming request/lead: headcount, date/time, budget, B2B flag, lead source, assigned KAM. A
    request can lead to one or more offers.
  </Card>
  <Card title="CateringOffer" icon="file-invoice">
    An offer for a request. Marked as *sent* via the `sent_to_customer_at` timestamp (not a status).
    An accepted offer becomes an Order.
  </Card>
  <Card title="Order" icon="receipt">
    The booked/paid transaction from an accepted offer. Carries the order status, the Stornogarantie line item,
    and the [post-booking checklist](/en/kam-handbook/post-booking-checklist).
  </Card>
  <Card title="Customer vs. Company (B2B)" icon="building">
    A request has a Customer and an `is_b2b` flag; Companies are a separate B2B entity (billing profiles,
    Company users, event formats). *The exact distinction beyond the B2B flag is not spelled out in the code —
    clarify with the team if it matters.*
  </Card>
</CardGroup>

## Caterer side

- **Retailer = Caterer** — a catering business on the marketplace.
- **Branch = caterer location** — a retailer can have multiple branches, each with its own opening hours,
  minimum order value, and delivery support.

## Key terms

- **Stornogarantie** (cancellation guarantee) — a tiered line item added automatically to offers/orders (SKU prefix
  `EGORA-STORNOGARANTIE`), in four tiers (T1–T4) that scale with order value. On an **offer** it's
  added/updated automatically unless the KAM opts out (`exclude_stornogarantie`); on an **order** it's
  only **adjusted**, never added automatically. The opt-out carries over from the offer to the order.
- **KAM** — Key Account Manager: owns a request end-to-end.
- **GMV** — Gross Merchandise Value (booked order value). The internal KPI dashboards exclude checkout,
  cancelled/storno, and payment-failed orders from booked GMV totals.

## Where eGora operates

**Berlin and Munich only** — the only cities configured in the system. Timezone: Europe/Berlin.
