## How it stays current

<Steps>
  <Step title="Versioned in Git">
    Every change is a reviewable pull request in `Schaufenster/docs` — nothing changes silently.
  </Step>
  <Step title="Shipped features automatically produce a changelog draft">
    Mintlify's changelog automation drafts an entry when product code changes — see
    [Release communication](/en/it/feature-changelog).
  </Step>
  <Step title="Claude always reads the latest state">
    Because the same source feeds Claude via MCP, the assistant always answers based on the current knowledge.
  </Step>
</Steps>
