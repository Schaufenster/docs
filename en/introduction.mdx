## How it stays current

<Steps>
  <Step title="Versioned in git">
    Every change is a reviewable pull request in `Schaufenster/docs` — nothing changes silently.
  </Step>
  <Step title="Shipped features auto-generate a changelog draft">
    Mintlify's changelog automation drafts an entry whenever product code changes — see
    [release communication](/it/feature-changelog).
  </Step>
  <Step title="Claude always reads the current state">
    Because the same source feeds Claude via MCP, the assistant always answers based on current knowledge.
  </Step>
</Steps>
