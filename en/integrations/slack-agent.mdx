With the Mintlify Slack agent, anyone can draft a KB update from inside Slack — without giving the agent
broad access to your messages.

## How it is scoped (privacy-friendly)

<CardGroup cols={2}>
  <Card title="@mention only" icon="at">
    Nothing happens unless you **@mention** the agent in a thread. It acts only in that thread.
  </Card>
  <Card title="No passive listening" icon="eye-slash">
    Channel "knowledge ingestion" is **off** (or restricted to a dedicated `#kb-updates` channel). It does
    not read DMs or all channels.
  </Card>
  <Card title="PR-gated" icon="code-pull-request">
    It opens a **pull request** against `Schaufenster/docs` — it never commits directly to `main`.
  </Card>
  <Card title="Preview links" icon="eye">
    Review the proposed change on a preview URL before merging.
  </Card>
</CardGroup>

## How to use it

<Steps>
  <Step title="Mention the agent">
    In a thread, type for example *"@mintlify document: KAMs can now set Du/Sie per customer in the backend."*
  </Step>
  <Step title="Review the PR">
    The agent drafts a docs change and opens a PR with a preview link.
  </Step>
  <Step title="Merge">
    If it looks right, merge — Mintlify redeploys and the KB (web + Claude MCP) updates.
  </Step>
</Steps>

<Warning>
  A Slack **workspace admin** approves the agent's permissions at install time. Review the requested scopes.
</Warning>
