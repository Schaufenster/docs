---
title: "Claude an die KB anbinden"
description: "Wie ein KAM Claude (Team Project, Code, Cowork) mit der Wissensdatenbank, Live-Daten und Notion verdrahtet."
icon: "robot"
---

Der eGora Team Assistant wird erst dann nützlich, wenn Claude die drei Live-Quellen lesen kann. Das Setup erfolgt pro Person.

<Tip>
  Eine vollständige, geführte Einrichtung aller Connectors (inkl. HubSpot) findest Du unter
  [Den eGora Team Assistant einrichten](/integrations/team-assistant-setup). Diese Seite ist die
  Kurzfassung für die KB-Anbindung.
</Tip>

<Steps>
  <Step title="Öffne das eGora Team Assistant Project">
    Öffne in claude.ai das geteilte Team Project. Power-User können auch Claude Code / Cowork verwenden.
  </Step>
  <Step title="Verbinde die KB (Mintlify MCP)">
    Füge einen **Custom Connector** hinzu, der auf den authentifizierten MCP-Endpoint der KB zeigt:
    `https://docs.egora-catering.de/mcp`. Melde dich bei Aufforderung als eGora-Org-Mitglied an.

    In Claude Code stattdessen:
    ```bash
    claude mcp add --transport http egora-kb https://docs.egora-catering.de/mcp
    ```
  </Step>
  <Step title="Verbinde Live-Daten (egora-data MCP)">
    Aktiviere den **egora-data**-Connector und authentifiziere dich. Siehe [egora data](/integrations/egora-mcp).
  </Step>
  <Step title="Verbinde Notion">
    Aktiviere den **Notion**-Connector, damit Claude Arbeitsdokumente live lesen kann — siehe [Notion](/integrations/notion).
  </Step>
</Steps>

<Tip>
  Sobald alles verbunden ist, frage zum Beispiel *"Wie onboarden wir einen neuen Account?"* (KB), *"Wie viele B2B-Anfragen
  in den letzten 7 Tagen?"* (egora-data) oder *"Was ist der aktuellste Entwurf des Eskalationsprozesses?"* (Notion).
</Tip>

<Note>
  Die Live-KB-Domain ist `docs.egora-catering.de`.
</Note>
