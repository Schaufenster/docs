The KB is **one git-backed source** (`Schaufenster/docs` → Mintlify). It is read by two kinds of consumer
and fed by a few reviewed inflows.

## Who reads it

<CardGroup cols={2}>
  <Card title="Humans (this site)" icon="user">
    KAMs read the web KB directly. Access is restricted to eGora org members.
  </Card>
  <Card title="Claude (via MCP)" icon="robot">
    The eGora KAM Assistant reads the KB through Mintlify's MCP server — see
    [Connecting Claude](/integrations/claude-setup).
  </Card>
</CardGroup>

## What feeds Claude

Claude is grounded by **four** sources, each for a different job:

| Source | What it's for |
| --- | --- |
| **Mintlify KB MCP** | Stable, official knowledge (this KB) |
| **egora-data MCP** | Live business data (orders, requests, KPIs) — see [egora data](/integrations/egora-mcp) |
| **Notion** | Working/in-flux docs, read live — see [Notion](/integrations/notion) |
| **Skills** | Power-user behaviors in Claude Code / Cowork |

Rule of thumb: **KB MCP** for "how does X work", **egora-data** for "what are the current numbers",
**Notion** for "what's the latest draft".

## What keeps it current (authoring inflows)

Every change lands as a **reviewable pull request** — nothing is committed silently.

  <Card title="Changelog automation" icon="code-branch" href="/integrations/changelog">

  </Card>
  <Card title="Slack agent" icon="slack" href="/integrations/slack-agent">
