Die ursprüngliche Lücke, auf die diese KB abzielt: **Features werden ausgeliefert, ohne kommuniziert zu werden** — an KAM, Kunden und
Caterer. Die Changelog-Automatisierung schließt sie.

## Funktionsweise (aktuelles Setup — Route 2, credit-frei)

Seit 2026-07-09 läuft der Changelog über den lokalen **`/kb-update`-Skill** in Claude Code
(im egora-Repo unter `.claude/skills/kb-update/`) — die nativen Mintlify-Automatisierungen
(„Aktualisierung aus Code-Änderungen" + „Changelog-Entwurf") sind abgeschaltet, weil sie
Agent-Credits pro Push verbrauchen.

<Steps>
  <Step title="Committen wie gewohnt">
    Feature im egora-Repo committen — optional mit `Changelog:`-Trailer (siehe unten).
  </Step>
  <Step title="/kb-update ausführen">
    Claude Code liest alle Commits seit dem letzten Sync-Marker (`.kb-sync.json` in diesem
    Repo), entwirft die Einträge direkt in
    [Release-Kommunikation](/it/feature-changelog) und passt betroffene KB-Seiten an.
    Übersprungene Commits werden mit Begründung gelistet.
  </Step>
  <Step title="Review, Push, Ankündigung">
    Diff prüfen, nach `main` pushen — Mintlify deployt in ~1–2 Minuten kostenlos. Der Eintrag
    erreicht damit die KAMs (Web), Claude (MCP) und den Slack-Agenten. Die von Claude
    mitgelieferte Slack-Ankündigung postest du selbst ins Team.
  </Step>
</Steps>

<Note>
  Kostenmodell: ein Push löst nur den kostenlosen Site-Rebuild aus — **keine
  Mintlify-Agent-Credits**. Credits fallen nur noch an, wenn jemand den Slack-Agenten
  aktiv @mentionst.
</Note>

## Hilf nach: der `Changelog:`-Trailer

Wenn ein Commit für Nutzer relevant ist, füge eine einzeilige Trailer-Zeile hinzu, damit die Automatisierung ein sauberes Signal hat:

```
feature for the salutation style

Changelog: Customers can now be addressed Du/Sie; KAM sets it per customer.
```

Das ist optional, schärft aber den Entwurf — und sorgt für eine saubere Historie, falls wir die Engine irgendwann wechseln.

## Das gewünschte Format

Jeder Eintrag sagt, **was sich geändert hat**, **wer betroffen ist** (KAM / Kunden / Caterer) und **was zu tun ist** —
siehe die Beispiele unter [Release-Kommunikation](/it/feature-changelog).

<Note>
  Historie: Route 1 (Mintlifys native Agenten, auf Trial-Credits) lief bis 2026-07-09. Der Wechsel
  auf Route 2 war von Anfang an als Fallback geplant — der Skill schreibt **dieselbe**
  Changelog-Datei, nachgelagert musste nichts angepasst werden.
</Note>
