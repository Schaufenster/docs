## Why this pilot exists

New features regularly ship without being documented or communicated clearly to **KAM, customers and
caterers**. A docs-as-code knowledgebase aims to fix that by making documentation:

<Steps>
  <Step title="Versioned in git">
    Docs live next to a clear review process — every change is a commit, reviewable like code.
  </Step>
  <Step title="Published for humans">
    A searchable site with an AI assistant, organised by audience.
  </Step>
  <Step title="Readable by AI">
    Mintlify auto-generates `llms.txt` and `llms-full.txt`, so the same source can ground the upcoming
    **Claude Team** "eGora KAM Assistant" — one source of truth for people *and* Claude.
  </Step>
</Steps>

<Tip>
  Switch between **English** and **Deutsch** using the language selector in the top bar — the German
  market is core to eGora, so i18n quality is one of the things this pilot is testing.
</Tip>
