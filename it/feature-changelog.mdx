Every feature that ships gets a changelog entry here, organised by the three audiences it may affect: **KAM**, **Customers**, and **Caterers**. Use this page to understand what changed, what you need to do differently, and where to find the full documentation.

## The pattern

Every shipped feature gets one changelog entry that answers three questions:

<Steps>
  <Step title="What changed (one sentence)">
    Plain language, no jargon — the behaviour a user would notice.
  </Step>
  <Step title="Who is affected">
    Tag the audiences: **KAM**, **Customers**, **Caterers**. If an audience is affected, the entry must
    say what *they* should do differently.
  </Step>
  <Step title="Where to learn more">
    Link to the relevant page in the customer or caterer tab, so the announcement and the reference doc
    stay in sync.
  </Step>
</Steps>

## Changelog

<Update label="2026-06" description="Per-customer salutation (Du / Sie)">
  Customers can now be addressed formally (**Sie**) or informally (**Du**) consistently across emails,
  the customer backend, surveys and PDFs.

  <CardGroup cols={3}>
    <Card title="KAM" icon="headset">
      Set the salutation style on the customer record. Default is informal (**Du**).
    </Card>
    <Card title="Customers" icon="user">
      Outgoing communication now matches the chosen tone — no action needed.
    </Card>
    <Card title="Caterers" icon="utensils">
      Not affected.
    </Card>
  </CardGroup>
</Update>

<Update label="2026-06" description="Order-level cancellation guarantee (Stornogarantie)">
  The cancellation-guarantee tier now follows the **order total** automatically, and KAM can opt a
  single order out. Frozen once the order is invoiced or paid.

  <CardGroup cols={3}>
    <Card title="KAM" icon="headset">
      New per-order opt-out toggle. The tier resizes on its own — no manual recalculation.
    </Card>
    <Card title="Customers" icon="user">
      The guarantee line on the offer reflects the current order size.
    </Card>
    <Card title="Caterers" icon="utensils">
      The Stornogarantie line no longer appears on the delivery sheet (Lieferschein).
    </Card>
  </CardGroup>
</Update>

<Update label="2026-06" description="Disable layered matching in caterer matching">
  KAM can switch caterer matching to a flat, alphabetical list of all published caterers to pick
  directly, bypassing the scoring layers.

  <CardGroup cols={3}>
    <Card title="KAM" icon="headset">
      New "disable layer matching" toggle in the matching tool.
    </Card>
    <Card title="Customers" icon="user">
      Not affected.
    </Card>
    <Card title="Caterers" icon="utensils">
      Not affected.
    </Card>
  </CardGroup>
</Update>
