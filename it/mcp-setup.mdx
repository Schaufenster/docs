This connects the **Claude Desktop** app to the eGora data server so you can ask questions about orders,
catering requests, companies and more — directly in chat, against live (PII-redacted) data.

<Note>
  Adapted from the internal guide `docs/mcp/setup-claude-desktop-ui.de.md`. The German version of this
  page is available via the language switcher.
</Note>

## Prerequisites

- **Claude Desktop** (Mac or Windows), reasonably up to date.
- A **personal access token (PAT)** for eGora — request it from Steffen / the KAM lead. Format
  `4|XXXX…`. **Never share your token.**

## Setup

<Steps>
  <Step title="Open Developer settings">
    In Claude Desktop go to **Claude → Settings…** (`⌘ ,`), then select **Developer** in the sidebar.
  </Step>
  <Step title="Edit the config">
    Under **Local MCP servers**, click **Edit config**. This opens `claude_desktop_config.json` in your
    default editor.
  </Step>
  <Step title="Add the egora-data server">
    Paste the block below. If the file already has an `mcpServers` object, add `egora-data` *inside* it
    rather than overwriting the whole file.

    <CodeGroup>
    ```json HTTP (recent versions)
    {
      "mcpServers": {
        "egora-data": {
          "type": "http",
          "url": "https://staging.egora-catering.de/mcp",
          "headers": {
            "Authorization": "Bearer YOUR_TOKEN_HERE"
          }
        }
      }
    }
    ```

    ```json mcp-remote (older versions, needs Node.js)
    {
      "mcpServers": {
        "egora-data": {
          "command": "npx",
          "args": [
            "-y",
            "mcp-remote",
            "https://staging.egora-catering.de/mcp",
            "--header",
            "Authorization: Bearer YOUR_TOKEN_HERE"
          ]
        }
      }
    }
    ```
    </CodeGroup>
  </Step>
  <Step title="Insert your token">
    Replace `YOUR_TOKEN_HERE` with your real token, including the `4|` prefix and the pipe character.
  </Step>
  <Step title="Restart Claude Desktop fully">
    Save the file (`⌘ S`), then **quit completely** with `⌘ Q` — closing the window is not enough, the
    process keeps running with the old config. Reopen Claude Desktop.
  </Step>
</Steps>

After restarting, **Settings → Developer → Local MCP servers** should show `egora-data` with a **green**
status.

## Test it

Open a new chat and try:

> *How many B2B catering requests did we have in the last 7 days, grouped by city?*

Claude should call the `list-catering-requests` tool and answer with real numbers. If Claude asks
whether it may use the connector — **allow it**.

## Production vs staging

| Staging | Production |
| --- | --- |
| `https://staging.egora-catering.de/mcp` | `https://egora-catering.de/mcp` |

Swap the URL and restart Claude Desktop.

## Security

<Warning>
  The token sits in **plain text** in `claude_desktop_config.json`. Never commit or sync this file
  (Dropbox, iCloud, OneDrive, git). If your laptop is lost or the file gets synced, have Steffen revoke
  the token immediately. Don't share tokens in Slack/email — request a separate token per person.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="'failed' right after opening">
    Token invalid or expired — request a new one from Steffen.
  </Accordion>
  <Accordion title="egora-data doesn't appear in the list at all">
    Broken JSON (missing comma, quote, …). Validate the file, e.g. at jsonlint.com.
  </Accordion>
  <Accordion title="'unknown transport type' or similar">
    Desktop version too old for `"type": "http"`. Use the **mcp-remote** variant above (requires
    Node.js: `brew install node` on Mac, or nodejs.org).
  </Accordion>
  <Accordion title="Tools visible, but answers are empty">
    The staging server may be temporarily unreachable — ping Steffen.
  </Accordion>
  <Accordion title="Connector is green, but Claude says it has no access">
    No tool permission granted in the current chat yet — click the 🔌 icon in the chat and enable the
    tools.
  </Accordion>
</AccordionGroup>
