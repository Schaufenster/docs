This connects **Claude** to the eGora data server so you can ask questions about orders, catering
requests, companies, KPIs and more — directly in chat, against live (PII-redacted) data.

<Note>
  There are **two** ways to connect, depending on which Claude you use. **OAuth is the default** for
  Claude on the web (Cowork / Teams) and is the simplest. The **token** method is only for Claude Code
  (CLI) and Claude Desktop power users.
</Note>

## Option A — OAuth (Claude web / Cowork / Teams) — recommended

No token to copy and no client to create — Claude registers itself and you just authorize.

### Prerequisites
- You are an **eGora Nova user** with the **`admin` role**. If not, ask Steffen to create your user and
  assign the role (Nova → Users).

### Connect

<Steps>
  <Step title="Add a custom connector">
    In Claude, go to **Settings → Connectors → Add custom connector**.
  </Step>
  <Step title="Paste the URL, leave the rest blank">
    URL: `https://egora-catering.de/mcp` — **leave Client ID and Client Secret empty.**
  </Step>
  <Step title="Connect & authorize">
    Click **Connect**. You're redirected to **log into eGora's Nova as yourself**, then click
    **Authorize**. Done — the connector is active.
  </Step>
</Steps>

### Test it
Open a chat and ask:

> *How many B2B catering requests did we have in the last 7 days, grouped by city?*

Claude should call `list-catering-requests` and answer with real numbers.

## Option B — Token (Claude Code / Desktop power users)

Claude Code (CLI) and older Claude Desktop versions use a **personal access token (PAT)** instead of
OAuth.

### Prerequisites
- A **personal access token** for eGora — request it from Steffen. Format `4|XXXX…`. **Never share it.**

### Claude Code
```bash
claude mcp add --transport http egora-data https://egora-catering.de/mcp \
  --header "Authorization: Bearer YOUR_TOKEN_HERE"
```

### Claude Desktop (config file)
**Settings → Developer → Local MCP servers → Edit config**, then add inside `mcpServers`:

<CodeGroup>
```json HTTP (recent versions)
{
  "mcpServers": {
    "egora-data": {
      "type": "http",
      "url": "https://egora-catering.de/mcp",
      "headers": { "Authorization": "Bearer YOUR_TOKEN_HERE" }
    }
  }
}
```

```json mcp-remote (older versions, needs Node.js)
{
  "mcpServers": {
    "egora-data": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote", "https://egora-catering.de/mcp",
        "--header", "Authorization: Bearer YOUR_TOKEN_HERE"
      ]
    }
  }
}
```
</CodeGroup>

Replace `YOUR_TOKEN_HERE` with your real token (including the `4|` prefix), save, and **fully quit**
Claude Desktop (`⌘ Q`) before reopening.

<Warning>
  The token sits in **plain text** in `claude_desktop_config.json`. Never commit or sync that file. If
  it leaks, have Steffen revoke the token immediately.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="OAuth: 'Authorize' page won't load or you can't log in">
    You need an eGora Nova user with the `admin` role — ask Steffen to set it up.
  </Accordion>
  <Accordion title="Token: 'failed' right after connecting">
    Token invalid or expired — request a new one from Steffen.
  </Accordion>
  <Accordion title="Token: 'unknown transport type'">
    Claude Desktop too old for `"type": "http"` — use the **mcp-remote** variant (needs Node.js).
  </Accordion>
  <Accordion title="Connected, but Claude says it has no access">
    No tool permission granted in the current chat yet — enable the connector's tools in the chat.
  </Accordion>
</AccordionGroup>
