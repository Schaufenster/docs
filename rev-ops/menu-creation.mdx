## Was ist Menu Creation?

Die Menü-Erstellung ist ein zentraler Prozessschritt zwischen Kundenanfrage und Angebot. Im Regelfall (Großteil der Anfragen) erstellt der KAM das Angebot selbst aus Caterer-Katalogen und historischen Angeboten. Bei komplexen/großen Anfragen erstellt der Caterer ein individuelles Angebot. Geschwindigkeit hier = direkte TTO-Impact = direkte CVR-Impact.

### Zwei Wege zur Caterer-Auswahl

1. **Caterer Matching Tool** — schlägt anhand der Anfrage-Parameter passende Caterer vor. *Hinweis: Das Tool befindet sich noch in der Anfangsphase und wird aktuell laufend optimiert.*
2. **Catering-Angebots-Tool** — Caterer wird entweder **manuell** ausgewählt oder über das **Historic Offer Tool** anhand der Kriterien der Anfrage aus einem ähnlichen historischen Angebot übernommen.

## AI-Tool-Stack (Custom, GPT-API-basiert)

| Tool | Was es tut |
|------|-----------|
| **Caterer Matching Tool** | Matcht Anfrage-Parameter mit passenden Caterern (separates Tool, nicht Notion Matching Base). Aktuell noch in der Anfangsphase, wird laufend optimiert. |
| **Catering-Angebots-Tool** | Manuelle Caterer-Auswahl beim Bauen des Angebots — alternativ kann darüber das **Historic Offer Tool** angestoßen werden, um den Caterer aus einem passenden historischen Angebot zu übernehmen. |
| **Historic Offer Tool** | Sucht anhand der Anfrage-Kriterien nach ähnlichen Angeboten aus der Vergangenheit und nutzt sie als Vorlage (inkl. Caterer-Auswahl). |
| **PDF-to-Offer Converter** | Konvertiert Caterer-PDFs ins egora-Angebotsformat |
| **Allergen AI** | Erkennt und taggt Allergene automatisch |
| **Description Optimizer AI** | Verbessert Menü-Beschreibungen automatisch |
| **Magic Menu Creation** | Konzept: vollautomatische Menüerstellung aus Anfrage-Parametern |

Alles custom-built, mit API-Anbindung an GPT. Läuft über das Admin Backend.

## Admin Backend

Laravel Nova — das zentrale Backend für KAMs und Operations. Hier werden Angebote erstellt, Deals gemanagt, PDFs generiert, Caterer-Kataloge gepflegt. Auch die AI-Tools laufen über dieses Backend.

## Caterer-Kataloge — Bottleneck

Unvollständige Kataloge zwingen KAMs zur manuellen Abstimmung mit Caterern → verlängert TTO → reduziert Skalierbarkeit. Je vollständiger der Katalog, desto mehr kann der KAM selbst abwickeln und desto schneller geht das Angebot raus. Direkte Verbindung zu → **Caterer/Supply** *(Department noch in Arbeit)*.

## Magic Menu Creation — Konzept

Ziel: Aus den Anfrage-Parametern (Budget, Personenzahl, Anlass, Präferenzen) automatisch ein vollständiges Menü mit Preisen generieren — ohne dass der KAM manuell zusammenstellen muss. Würde TTO für Standard-Anfragen drastisch drücken und KAM-Kapazität für komplexe Deals freisetzen.
