## Architektur-Überblick

```
Kunde → Webflow (Website/Form) → Admin Backend (ERP) → HubSpot (CRM)
                                       ↕                    ↕
                                    Stripe (Billing)    Marketing Automation
                                       ↕
                              Customer Backend (Lovable)
                              Caterer Backend (Lovable)
```

**Datenfluss:**
1. Kunde füllt Request Form auf Webflow aus
2. Anfrage landet im Admin Backend + HubSpot (Deal Pipeline)
3. KAM bearbeitet im Admin Backend (Menü, Caterer-Matching)
4. Angebot wird aus Admin Backend generiert → an Kunden
5. Bei Buchung: Rechnung über Admin Backend → Stripe
6. Kunde sieht alles im Customer Backend (Lovable)
7. Caterer sieht Aufträge im Caterer Backend (Lovable)

## Stärken des Stacks

- **HubSpot-Integration:** Detailliert aufgebaut, Lifecycle Stages automatisch, Potenzialschätzung
- **Admin Backend als ERP:** Zentrales Werkzeug für den operativen Kern (KAM-Arbeit)
- **AI im Produkt:** Allergen Tagging, Menu Creation, PDF-to-Offer live
- **Stripe-Anbindung:** Billing automatisiert über Admin Backend

## Grundsatz

Team ist agil beim Tool-Stack — offen für Optimierung und Konsolidierung, keine emotionale Bindung an Tools. Wechsel wenn es Sinn macht.
