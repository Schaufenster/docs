Skills erweitern den eGora-KI-Assistenten um feste Fähigkeiten und Workflows. Sie sind **Capability-Config**,
kein Wissens-Content: die **lauffähigen Kopien** leben im Claude-Teams-Projekt bzw. in Claude Code / Cowork.
**Diese Seiten sind die Dokumentation der Wahrheit** (Versionsstand + voller Inhalt), damit nichts verloren geht.

<Info>
  **Quelle der Wahrheit:** diese Mintlify-Seiten. **Runtime-Kopien:** Claude-Teams-Projekt (Chat-Skills) bzw.
  Claude Code / Cowork. Bei einer Änderung **beide** aktualisieren.
</Info>

## Runtime-Legende

- **Claude Teams (Chat)** — reine Prompt-Skills, laufen direkt im Teams-Projekt-Chat.
- **Cowork / Scheduled Agent** — brauchen Connectoren (Slack / Asana) + persistenten State, laufen scheduled.
- **Claude Code / Cowork** — brauchen Filesystem, Subagents, MCP-Mounts.
- **Code-Execution** — brauchen Python / Code-Sandbox.

## In Claude Teams aktiv (7)

Die sieben reinen Chat-Skills sind im Teams-Projekt hinterlegt und triggern per natürlicher Sprache:
**Challenge Me, Take a Step Back, Offer-Mail Assistant, Request-Info Structurer, Ops Lead, Project Manager,
LinkedIn Post**. (LinkedIn Post zieht echte Zahlen besser, wenn die HubSpot- / Notion- / Asana-Connectoren
am Projekt hängen.)

## Separates Setup (4)

Diese laufen **nicht** im Teams-Chat — sie brauchen Filesystem, Connectoren mit State oder Code-Execution
und gehören nach **Claude Code bzw. Cowork**: **Get Shit Done, Daily Briefing, Daily Debrief, egora Branding**.
Voll dokumentiert sind sie hier trotzdem (kein Informationsverlust), inkl. der gebündelten Referenzen und des
Python-Helpers bei egora Branding.
