## Brand essence

**Three things define every egora document:**

1. **Black is the brand-anchor.** Most documents are black-background — that's the egora signature, inherited from the team-photo style. Light/cream backgrounds are reserved for content-dense knowledge docs or saisonale Broschüren where readability of long prose matters more.
2. **`#5E4FF0` is the only accent color.** This purple does headlines, highlights, big numbers, accent strips, bullet points. Don't add a second accent — semantic colors (green/yellow/red for status) are separate from brand-color.
3. **Anton for headlines, Inter for everything else.** Anton is condensed, brutalist, all-caps-feeling — it carries the energy. Inter does body text, labels, anything that needs to be readable at small sizes.

If those three are in place and the layout doesn't overlap, it'll feel right.

## Working method

1. **Read** `references/colors-and-typography.md` first
2. **Pick** the matching layout pattern in `references/layout-patterns.md` and read just that section
3. **Copy** the helper code from `references/helper-code.md` (or import `scripts/egora_style.py` directly)
4. **Build** the document — slide by slide if it's a deck, page by page if it's a doc
5. **Render** each page to PNG and visually QA — check the QA checklist in `references/qa-checklist.md`
6. **Fix** any text overlap, footer collision, or truncation before sharing
7. **Save** the final PDF to the outputs folder with a clear name

The QA step is where most builds fail. Don't skip it. Use the checklist.

## Sprachregel — language rule

This is what tripped earlier builds. Read carefully.

### The rule

- **Headlines (Anton):** English, short, punchy. Examples: "RECURRING IS WINNING.", "PROFITABLE GROWTH.", "WIR BAUEN ANDERS." — German is fine if it's a single short statement.
- **Body and sublines:** German throughout. Denglisch words are OK (GMV, Recurring, Pipeline, AI, Hires, Backend, Forecast, Customer-Backend, Account Manager, etc.) but never mix German and English **inside one sentence**.
- **Customer proposals:** Use customer's language end-to-end. G2 Esports asked for English → entire document in English. If unsure, ask.

### How to test it

For every body sentence ask: is this whole sentence German, or whole English? If it's mixed, rewrite.

**Bad (mixed in one sentence):**
> "AI-First is our Operating Mode — kein Buzzword, sondern unsere täglich angewandte Velocity."

**Good (consistent German with denglisch terms):**
> "AI-First ist Operating Mode — kein Buzzword. Claude Code, Lovable und eigene Agents im Daily Workflow."

**Also good (consistent English):**
> "AI-First is our operating mode — not a buzzword. Claude Code, Lovable, and custom agents in daily workflow."

## Pattern A — Investor / Quartals-Reporting

**Use for:** Quarterly investor updates, board materials, fundraising-prep PDFs.

**Format:** A4 landscape, 6-7 slides, black background dominant.

**Tone:** Bullish but not loud. Show asymmetry (3× growth + profitable). Misses brief and honest, never hidden.

### Slide architecture

1. **Hero (Cover)** — Mega "Big Number" + headline, 4 KPI tiles, main chart in white card
2. **Q-Numbers Deep Dive** — Stacked chart NC vs BK (recurring story), mini-stats on the side
3. **Building Different** — Backends/AI/velocity, 3 cards with backend screenshots
4. **Event / Community** (optional) — Taste & Connect-style photo grid + impact stats
5. **Outlook** — Plan-chart + 4 strategic pillars
6. **Profitable Growth** — EBIT YoY chart prominent + story bullets + "The Ask" sidebar
7. **Team + Logo Wall** — Team photos (1:1 square) + customer logos in white card

### Required content for investor docs

Always include:
- **YoY comparison** (cohort thinking)
- **Recurring share** (marketplace-health indicator)
- **Profitability metric** (EBIT %, not burn — egora is profitable since Feb 2026)
- **Team size** (capital efficiency signal — currently 6 internal)

### Hero-slide composition (Slide 1)

- Mega Big Number (Anton 80-92pt purple) at left-top, e.g. "3,06×"
- Sub-line right of or below (Anton 18-30pt white): "YOY GROWTH IN Q1. APRIL PLAN GETROFFEN."
- 4 KPI tiles in a row below: each `LIGHT_BG` rounded card, 70pt tall, with Anton 28-32pt purple number + label + sub
- Main chart in full-width white card at bottom, ~200pt tall

### Charts on dark background

White card with rounded corners (`WHITE_CARD` fill, 6-8pt radius), Anton 11-12pt black for chart label, Inter 9pt dark-gray for sub-label, chart embedded with ~12pt padding to edges.

## Pattern B — Customer-Proposal

**Use for:** Customer-specific proposals (G2 Esports, new accounts, daily-lunch setups, RFPs).

**Format:** A4 landscape, 8-10 slides, black background.

**Tone:** "We listened" — quote original requirements as proof. Performance-focused, precise, low marketing-speak. Use customer's language end-to-end.

### Slide architecture

1. **Cover** — `[CUSTOMER] × egora` typography. Customer name in Anton 80pt white, "× egora" in purple. Subtitle. "Prepared for / Prepared by" sections at bottom with names and contacts. Date in footer.
2. **What We Heard** — "We listened carefully" headline + setup at a glance (3 KPI tiles) + 4 tags + original email quote as proof
3. **How It Works** — 4 steps cards (One Briefing · We Curate · Single Touchpoint · It Gets Better)
4. **Menus / Details** — 1-N slides with full content. **Never summarize when user asks for complete listings** — show every dish, every component, every option.
5. **Pricing & Add-Ons** — 4 price tiles + add-ons section in white card
6. **150+ Caterers · One Network** — cuisine variety + theme weeks
7. **Why It Works** — 4 reasons (less vendor mgmt, one touchpoint, minimal team effort, fast adjustments) + 1-2 testimonials
8. **Next Steps + Contact** — trial week + contact details

### Cover-slide composition

- Top bar: small `egora` wordmark + uppercase "DAILY LUNCH & DINNER PROPOSAL" (or similar)
- Title block at vertical center: customer name and "× egora" stacked
- Subtitle below in Inter SemiBold gray
- Bottom strip: 2-column "Prepared for" / "Prepared by" with names + contact info
- Footer center: "MAY 2026" (or month) in Anton 14pt gray

### What We Heard slide (Slide 2)

This is the "we listened" proof. Include the original email quote from the customer (or their requirements quoted verbatim) in a purple box at the bottom. This is what makes the proposal feel personal rather than templated.

### Menu slides

When showing menus or component lists, **show everything**. If customer asked for a 7-day weekly menu with 4 categories per day, show all 28 cells with real dishes. Never show "Highlights" instead of the full list.

For data tables (menus, packages, items), use a structured table with alternating row backgrounds (`#F9FAFB`) and clear category labels.

**Use for:** Monthly recaps, team updates, internal celebrations.

**Format:** A4 landscape, 6-7 slides, black background.

**Tone:** Pride + hype. Celebrate wins loud. Misses framed as next-month lessons. Personal shoutouts only where organic.

### Slide architecture

1. **Hero** — "APRIL CRUSHED IT" / "MAY DELIVERED" headline + 4 KPI tiles + hero chart
2. **Progression** — Multi-period bar chart (e.g., 16 months Jan 2025 → April 2026) showing growth-over-time. Gray bars for past year, purple for current year. + 3 killer-stat tiles below.
3. **Sources Deep Dive** — Donut chart (NC vs BK) + top-source highlights + insight banner
4. **KR Status Tracker** — 4-5 KR cards with 🟢🟡🔴 status strips
5. **Wins** — 6 cards in 3×2 grid: product wins, personal shoutouts, partnership wins
6. **Outlook / Hebel** — 3-4 strategic levers (not task-level) + bottom banner with hype message
7. **WIR** (closing) — "BUILT BY 6 / TRUSTED BY THE BEST" + team photo strip (square 1:1 cells) + closing statement ("LET'S GO MAI.")

### Progression chart

When showing growth-over-time:
- Multi-period bar chart with year boundary marked
- Past year in `DARK_GRAY`, current year in `PURPLE`
- YoY multipliers (`2.8×`, `3.1×` etc) in Anton above the current-year bars
- Vertical divider at year boundary with year label

### Wins cards (3×2 grid)

6 cards. Each has:
- Top: small uppercase badge (e.g., "PRODUCT WIN", "PERSONAL WIN", "PIPELINE WIN")
- Title in Anton 18pt
- Body in Inter 9pt, 3 short lines

Personal shoutouts are fine if organic. Don't force every win to have a name.

## Pattern E — Knowledge-Doc / Internal

**Use for:** CVR-analyses, rev-ops findings, AM-cheatsheets, onboarding materials, internal knowledge transfer.

**Format:** A4 portrait, 2-6 pages depending on density. **Use white BG for dense data tables** (CVR analyses), **use black BG for energetic onboarding cheatsheets** (Hanna-style).

**Tone:** Factual, dense. For data: statistically-grounded findings only. Caveats explicit. TL;DR up front. For onboarding: clear, actionable, visualize stages.

### Knowledge-doc on white BG (data-heavy)

For CVR-style analyses with lots of tables:

1. **TL;DR** — 5 punchy statements at the top, one screen height max
2. **Datenbasis** — sample sizes, methodology in a clean box
3-N. **Findings by topic** — H2 sections with structured data tables. Use alternating row colors. Highlight key cells in purple + bold.
Final. **Methodische Caveats** — explicit list of what's NOT statistically robust

White background, purple H2 section headers with small underline accent, data tables with purple headers.

### Onboarding cheatsheet on black BG (energy)

For new-hire onboarding (Hanna-style AM-cheatsheet):

### references/qa-checklist.md

# QA Checklist

Run this checklist on every PDF before sharing. Render each page at 200+ dpi and visually check.

## Mandatory checks

These break the document if missed. Do every time.

- [ ] **Language consistency** — every body sentence is either complete German or complete English, never mixed. (Denglisch single words like "GMV", "Pipeline" are fine.)
- [ ] **Cache-bug fix in image handling** — temp filenames are path-hashed (`/tmp/crop_{md5(path)[:12]}.jpg`), not based on basename. Test: if any document has multiple images with the same basename across folders, they should render distinctly.
- [ ] **Footer clearance** — no chart, card, or content bleeds into the footer area. Footer divider sits at `M + 12pt`. Any content stops at `M + 28pt` minimum to leave breathing room.
- [ ] **Headline glyph spacing** — when using Anton at 80pt+, leave ≥75pt line spacing between consecutive headline lines. Glyphs collide otherwise.
- [ ] **Page header consistency** — every page except the cover has the standard `egora / SECTION / page X/Y` header with divider line.
- [ ] **Per-page HD QA** — render every page at 200+ dpi to PNG, then look at each. Most layout bugs (text truncation, cell overflow, overlapping) only show up there, not in the build code.

## Content checks

- [ ] **Vollständigkeit** — when the user asked for full listings (menus, components, line items), every item is there. Never show "Highlights" when "complete" was requested. Lesson from Sommerfest-Broschüre: full menu per page, not summary.
- [ ] **Pricing always with format context** — never just "€48". Always "ab €48 p.P. · je nach Gästezahl und Setup" or similar context line.
- [ ] **Customer proposals include email-quote** — Slide 2 ("What We Heard") should quote the customer's original requirements verbatim as proof.
- [ ] **Investor docs include the four mandatory metrics** — YoY comparison, Recurring share, EBIT %, Team size.
- [ ] **All-Hands include period-over-period progression** — Don't just show current period; show the trajectory.

## Layout checks

- [ ] **KPI tiles aligned and equal width** — when there are 4 in a row, gaps are equal and widths are equal. No "almost equal" — calculate exactly with `(PAGE_W - 2*M - (n-1)*gap) / n`.
- [ ] **Card rounded corners consistent** — pick 4, 5, 6, or 8pt and use the same across one slide. Don't mix.
- [ ] **Table cells fit the text** — render and check. If cells are truncated, increase column width, decrease font size, or wrap to multiple lines.
- [ ] **Status colors used semantically** — Green = positive/live, Yellow = at-risk/in-progress, Red = miss/off-track/end-state. Don't use status colors as decoration.
- [ ] **Purple used as accent, not as fill-everywhere** — purple is the only accent. Don't fill 4 things purple in a row. The eye needs rest.

## Typography checks

- [ ] **Anton only for headlines and big numbers** — never as body text (it's hard to read at small sizes)
- [ ] **Inter at 8pt minimum** — anything smaller is unreadable when printed
- [ ] **Body text at 9-11pt** — too small fatigues the reader, too big looks juvenile
- [ ] **Section headings clearly bigger than body** — visual hierarchy needs to be obvious at a glance

## Brand-feel checks

- [ ] **The document feels like egora at first glance** — black BG, purple accents, Anton headlines. If a stranger looked at it for 1 second, would they recognize it as egora?
- [ ] **Tone matches use case** — investor doc isn't hype-bro, all-hands isn't cold-corporate
- [ ] **No marketing fluff** — "innovative cutting-edge solutions" / "leveraging synergies" / "best-in-class platform" — none of that
- [ ] **No engagement bait** — "Want to know more?" / "Tag a colleague who..." / "🚀🚀🚀"
- [ ] **No false vulnerability** — "We almost gave up..." / "It was hard, but..."

## Customer-proposal extra checks

- [ ] **Customer name appears prominently** — in the cover and page headers (e.g., `egora × G2 ESPORTS / SECTION`)
- [ ] **Language matches customer's preference** — German for German customers, English for English-speaking
- [ ] **"We listened" proof present** — email quote, original requirements, named contact
- [ ] **Contact details correct** — sender name, email, phone

## Investor doc extra checks

- [ ] **The Ask is explicit but not pushy** — clear contact path, no hard close
- [ ] **Misses are framed, not buried** — if you have them, explain in 1-2 lines
- [ ] **Charts are honest** — no clipped y-axes that exaggerate moves

## Sommerfest brochure extra checks

- [ ] **Every menu page has the same structure** — title, format, mood, sections, price block. Consistency is the brand here.
- [ ] **Saisonale accents are subtle** — coral and sage are seasoning, not main colors. Don't decorate every corner.
- [ ] **Reply-trigger on every menu page** — CTA "Antworte mit der Menü-Nummer" present.

## Knowledge-doc extra checks

- [ ] **TL;DR fits on one screen** — if not, it's not a TL;DR
- [ ] **Sample sizes shown for every statistic** — `n=1612`, `n=8`, etc.
- [ ] **Caveats explicit** — what's NOT robust must be stated
- [ ] **Tables have clear headers** — column meaning obvious from header alone

## Final check before sharing

- [ ] **Save to user's egora workspace** (`egora Strategy & Goalsetting/`), not just temp folder
- [ ] **Provide `computer://` link** for the user to open the file directly
- [ ] **Write a brief summary** of what was built, with section headers
- [ ] **Offer iteration** — "If something needs adjusting (X, Y), tell me. Otherwise: done."

If anything in this list isn't right, fix it before sharing. Especially the mandatory checks. A document that fails on language consistency or has chart-footer collisions makes egora look amateur.

## Core drawing helpers

```python
def fill_rect(c, x, y, w, h, color):
    """Fill a rectangle without stroke."""
    c.setFillColor(color)
    c.rect(x, y, w, h, fill=1, stroke=0)


def text(c, x, y, s, font='Inter', size=10, color=WHITE, anchor='left'):
    """Draw text with anchor (left, right, center)."""
    c.setFont(font, size)
    c.setFillColor(color)
    if anchor == 'left':
        c.drawString(x, y, s)
    elif anchor == 'right':
        c.drawRightString(x, y, s)
    elif anchor == 'center':
        c.drawCentredString(x, y, s)


def wrap_text(s, max_chars):
    """Word-wrap helper for body text. Returns list of lines."""
    words = s.split(' ')
    lines = []
    line = ''
    for w in words:
        test = (line + ' ' + w).strip()
        if len(test) <= max_chars:
            line = test
        else:
            if line:
                lines.append(line)
            line = w
    if line:
        lines.append(line)
    return lines
```

## Image embedding — with path-hash temp fix

This is critical. If you don't path-hash the temp filenames, images with the same basename across different folders (e.g., `customer-backend/1.jpg` + `caterer-backend/1.jpg`) will overwrite each other and you'll get the same screenshot three times.

```python
def draw_image_fit(c, path, x, y, w, h, mode='cover'):
    """Draw image fitted into box. mode='cover' fills box (crops),
    'contain' fits inside (letterboxes)."""
    img = PILImage.open(path)
    iw, ih = img.size
    iar = iw / ih
    bar = w / h
    if mode == 'cover':
        if iar > bar:
            new_w = ih * bar
            crop = (int((iw - new_w) / 2), 0, int((iw + new_w) / 2), ih)
        else:
            new_h = iw / bar
            crop = (0, int((ih - new_h) / 2), iw, int((ih + new_h) / 2))
        img = img.crop(crop)
        # CRITICAL: path-hash temp filename to avoid collisions
        path_hash = hashlib.md5(path.encode()).hexdigest()[:12]
        tmp = f'/tmp/crop_{path_hash}.jpg'
        if img.mode != 'RGB':
            img = img.convert('RGB')
        img.save(tmp, 'JPEG', quality=88)
        c.drawImage(tmp, x, y, w, h, mask='auto')
    else:
        # contain mode — no crop
        if iar > bar:
            dw = w
            dh = w / iar
            dx = x
            dy = y + (h - dh) / 2
        else:
            dh = h
            dw = h * iar
            dx = x + (w - dw) / 2
            dy = y
        c.drawImage(path, dx, dy, dw, dh, mask='auto', preserveAspectRatio=True)
```

    c.setStrokeColor(BORDER)
    c.setLineWidth(0.5)
    c.line(m, m + 12, page_w - m, m + 12)
    text(c, m, m, label, font='Inter', size=8, color=MID_GRAY)
    text(c, page_w - m, m, date_str, font='Inter', size=8, color=MID_GRAY, anchor='right')

    """Number + UPPERCASE title with purple accent line below."""
    text(c, x, y, num, font='Anton', size=18, color=PURPLE)
    text(c, x + 30, y, title.upper(), font='Anton', size=16, color=WHITE)

## Arrow drawing for diagrams

```python
def draw_arrow(c, x1, y1, x2, y2, color, width=1.2, head_size=5, style='solid'):
    """Draw a directional arrow with optional dashed style."""
    c.setStrokeColor(color)
    c.setLineWidth(width)
    if style == 'dashed':
        c.setDash(3, 2)
    else:
        c.setDash()
    c.line(x1, y1, x2, y2)
    c.setDash()
    # Arrow head
    angle = math.atan2(y2 - y1, x2 - x1)
    ax1 = x2 - head_size * math.cos(angle - math.pi / 6)
    ay1 = y2 - head_size * math.sin(angle - math.pi / 6)
    ax2 = x2 - head_size * math.cos(angle + math.pi / 6)
    ay2 = y2 - head_size * math.sin(angle + math.pi / 6)
    c.setFillColor(color)
    p = c.beginPath()
    p.moveTo(x2, y2)
    p.lineTo(ax1, ay1)
    p.lineTo(ax2, ay2)
    p.close()
    c.drawPath(p, fill=1, stroke=0)
```

def fill_rect(c, x, y, w, h, color):
    """Fill a rectangle without stroke."""
    c.setFillColor(color)
    c.rect(x, y, w, h, fill=1, stroke=0)


def text(c, x, y, s, font='Inter', size=10, color=WHITE, anchor='left'):
    """Draw text. anchor = 'left' | 'right' | 'center'."""
    c.setFont(font, size)
    c.setFillColor(color)
    if anchor == 'left':
        c.drawString(x, y, s)
    elif anchor == 'right':
        c.drawRightString(x, y, s)
    elif anchor == 'center':
        c.drawCentredString(x, y, s)


def wrap_text(s, max_chars):
    """Word-wrap text to a list of lines, each at most max_chars long."""
    words = s.split(' ')
    lines = []
    line = ''
    for w in words:
        test = (line + ' ' + w).strip()
        if len(test) <= max_chars:
            line = test
        else:
            if line:
                lines.append(line)
            line = w
    if line:
        lines.append(line)
    return lines


def draw_image_fit(c, path, x, y, w, h, mode='cover'):
    """Draw image fitted into box.

    mode='cover' fills box (crops if aspect differs)
    mode='contain' fits inside box (letterboxes)

    Uses path-hash temp filenames to avoid collisions when multiple
    images share a basename across different folders.
    """
    from PIL import Image as PILImage
    img = PILImage.open(path)
    iw, ih = img.size
    iar = iw / ih
    bar = w / h
    if mode == 'cover':
        if iar > bar:
            new_w = ih * bar
            crop = (int((iw - new_w) / 2), 0, int((iw + new_w) / 2), ih)
        else:
            new_h = iw / bar
            crop = (0, int((ih - new_h) / 2), iw, int((ih + new_h) / 2))
        img = img.crop(crop)
        path_hash = hashlib.md5(path.encode()).hexdigest()[:12]
        tmp = f'/tmp/crop_{path_hash}.jpg'
        if img.mode != 'RGB':
            img = img.convert('RGB')
        img.save(tmp, 'JPEG', quality=88)
        c.drawImage(tmp, x, y, w, h, mask='auto')
    else:
        if iar > bar:
            dw = w
            dh = w / iar
            dx = x
            dy = y + (h - dh) / 2
        else:
            dh = h
            dw = h * iar
            dx = x + (w - dw) / 2
            dy = y
        c.drawImage(path, dx, dy, dw, dh, mask='auto',
                    preserveAspectRatio=True)

def section_header(c, x, y, num, title):
    """Section header: NUMBER · UPPERCASE TITLE with purple underline.

    Returns the y position just below the header for continued content.
    """
    text(c, x, y, num, font='Anton', size=18, color=PURPLE)
    text(c, x + 30, y, title.upper(), font='Anton', size=16, color=WHITE)
    fill_rect(c, x, y - 6, 50, 1.5, PURPLE)
    return y - 18


def draw_arrow(c, x1, y1, x2, y2, color, width=1.2, head_size=5, style='solid'):
    """Draw an arrow with directional head.

    style='solid' or 'dashed'
    """
    c.setStrokeColor(color)
    c.setLineWidth(width)
    if style == 'dashed':
        c.setDash(3, 2)
    else:
        c.setDash()
    c.line(x1, y1, x2, y2)
    c.setDash()
    angle = math.atan2(y2 - y1, x2 - x1)
    ax1 = x2 - head_size * math.cos(angle - math.pi / 6)
    ay1 = y2 - head_size * math.sin(angle - math.pi / 6)
    ax2 = x2 - head_size * math.cos(angle + math.pi / 6)
    ay2 = y2 - head_size * math.sin(angle + math.pi / 6)
    c.setFillColor(color)
    p = c.beginPath()
    p.moveTo(x2, y2)
    p.lineTo(ax1, ay1)
    p.lineTo(ax2, ay2)
    p.close()
    c.drawPath(p, fill=1, stroke=0)

def kpi_tile(c, x, y_top, w, h, big_number, label, sub_text,
             big_color=PURPLE, fill_color=LIGHT_BG):
    """Standard KPI tile on dark BG.

    big_number: Anton 28-32pt
    label: Inter-Bold 10pt
    sub_text: Inter 8.5pt
    """
    c.setStrokeColor(BORDER)
    c.setFillColor(fill_color)
    c.setLineWidth(0.5)
    c.roundRect(x, y_top - h, w, h, 6, fill=1, stroke=1)
    text(c, x + 14, y_top - 32, big_number, font='Anton', size=28, color=big_color)
    text(c, x + 14, y_top - 50, label, font='Inter-Bold', size=10, color=WHITE)
    text(c, x + 14, y_top - 62, sub_text, font='Inter', size=8.5, color=GRAY)


def status_card(c, x, y_top, w, h, badge, title, body_lines,
                status_color=PURPLE, accent_position='top'):
    """Status card with colored accent and bullet body.

    accent_position: 'top' or 'left'
    body_lines: list of strings, each rendered with purple bullet
    """
    c.setStrokeColor(BORDER)
    c.setFillColor(LIGHT_BG)
    c.setLineWidth(0.5)
    c.roundRect(x, y_top - h, w, h, 7, fill=1, stroke=1)
    if accent_position == 'top':
        c.setFillColor(status_color)
        c.roundRect(x, y_top - 5, w, 5, 0, fill=1, stroke=0)
    elif accent_position == 'left':
        fill_rect(c, x, y_top - h, 5, h, status_color)
    text(c, x + 16, y_top - 22, badge, font='Inter-Bold', size=8, color=status_color)
    text(c, x + 16, y_top - 42, title, font='Anton', size=18, color=WHITE)
    by = y_top - 62
    for line in body_lines:
        wrapped = wrap_text(line, 38)
        for j, wl in enumerate(wrapped):
            if j == 0:
                text(c, x + 16, by, "·", font='Inter-Bold', size=11, color=status_color)
                text(c, x + 24, by, wl, font='Inter', size=9, color=GRAY)
            else:
                text(c, x + 24, by, wl, font='Inter', size=9, color=GRAY)
            by -= 12

def chart_clean(ax):
